# ST-007 — Implement authentication

**Story key:** `feature.authentication`

## Description

Add registration, login, logout and session management.

## Acceptance Criteria

- [ ] Users can register with email and password
- [ ] Users can log in and receive a valid session
- [ ] Users can log out and have their session invalidated
- [ ] Protected routes return 401 for unauthenticated requests
- [ ] Passwords are hashed before storage
- [ ] Payload Users collection has auth enabled
- [ ] Database migration is generated and executed for all schema changes (run migrate:create then migrate)

## Scope Boundaries

- Do NOT implement OAuth or social login in this story
- Do NOT implement password reset or email verification
- Do NOT implement MFA

## Expected Files

- `src/lib/auth.ts`
- `src/collections/Users.ts`
- `src/app/(auth)/login/page.tsx`

## Dependencies

- `bootstrap.backend`
- `bootstrap.frontend`

## Validation

- Run: `npm run build`
- Run: `npm test`
- Manual: Can register, log in, and access protected route
