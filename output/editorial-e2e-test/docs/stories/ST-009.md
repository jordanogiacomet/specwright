# ST-009 — Implement media library

**Story key:** `feature.media-library`

## Description

Allow uploading, listing, and managing media assets.

## Acceptance Criteria

- [ ] Users can upload images and files
- [ ] Uploaded media is stored and retrievable via URL
- [ ] Media list view shows uploaded assets with thumbnails
- [ ] File size and type validation is enforced on upload
- [ ] Database migration is generated and executed for all schema changes (run `npm run db:migrate:create` then `npm run db:migrate`)
- [ ] Payload Media collection uses the local filesystem upload adapter (store uploads in a `media/` directory at the project root)
- [ ] The local filesystem path is treated as an adapter boundary so object storage can be introduced later without changing callers
- [ ] Payload Media collection is configured with upload settings

## Scope Boundaries

- Do NOT redefine the CMS schema here — extend the existing `media` collection from the CMS content model story with upload/storage behavior only
- Do NOT implement image optimization or CDN delivery in this story
- Do NOT implement media usage tracking across content
- All migrations MUST be created in `src/lib/migrations/` using `npm run db:migrate:create` — do NOT create migration files in any other directory (e.g. `src/models/migrations/`)
- Do NOT configure S3 or external object storage — use Payload's local disk adapter for now

## Expected Files

- `src/collections/Media.ts`
- `src/payload.config.ts (updated with upload and storage settings)`
- `.env.example (updated with storage configuration)`

## Dependencies

- `feature.authentication`
- `product.cms-content-model`

## Validation

- Run: `npm run build`
- Run: `npm test`
- Manual: Can upload an image or document, retrieve its URL, and see validation reject disallowed file types.
