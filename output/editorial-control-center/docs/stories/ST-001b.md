# ST-001b тАФ Define CMS content model (part 2 of 2)

**Story key:** `product.cms-content-model-part-2`

## Description

Define CMS content model тАФ collections: pages, posts, authors, media; globals: site-settings, homepage.

## Acceptance Criteria

- [ ] `authors` can reference `media` for avatar/headshot assets.
- [ ] Payload collection `media` exists with fields `alt_text`, `caption`, `file`, `mime_type`, `size` тАФ Manage uploaded images and downloadable files.
- [ ] Payload global `site-settings` exists with fields `site_name`, `navigation`, `footer`, `default_seo` тАФ Store navigation, SEO defaults, and shared site metadata.
- [ ] Payload global `homepage` exists with fields `hero`, `featured_posts`, `featured_pages`, `seo` тАФ Store homepage hero content and featured editorial slots.
- [ ] Media requirements are explicit: support image, document uploads in the `media` collection.
- [ ] Only the collections/globals listed in this story are created; no extra editorial schema is invented outside the spec.
- [ ] Content editors can create and edit entries for each modeled collection/global through the admin surface.
- [ ] Database migration is generated and executed for all schema changes (run `npm run db:migrate:create` then `npm run db:migrate`)
- [ ] All collections and globals are registered in `src/payload.config.ts` using dedicated definition files.

## Scope Boundaries

- Do NOT implement draft/publish transitions, preview mode, or scheduled publishing in this story.
- Do NOT build media library UI or public page routes here тАФ only define the schema and registration points.
- Do NOT add collections/globals beyond the ones named in this story unless `spec.json` is updated first.

## Expected Files

- `src/collections/Media.ts`
- `src/globals/SiteSettings.ts`
- `src/globals/Homepage.ts`

## Dependencies

- `product.cms-content-model`

## Validation

- Run: `npm run build`
- Manual: Admin can create sample page/post entries, assign author/media relations, and edit globals without missing fields.
