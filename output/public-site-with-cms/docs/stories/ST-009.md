# ST-009 — Setup backend service

**Story key:** `bootstrap.backend`

## Description

Initialize backend using payload with database connection.

## Acceptance Criteria

- [ ] Backend is initialized using payload
- [ ] Server starts and responds to health check
- [ ] API is accessible at the configured port
- [ ] Payload config file exists with database adapter configured
- [ ] Payload admin panel is accessible at /admin

## Scope Boundaries

- Do NOT implement authentication yet
- Do NOT create domain collections or models yet
- Do NOT add business logic — only the backend skeleton

## Expected Files

- `src/payload.config.ts`
- `src/server.ts or src/app/(payload)/layout.tsx`

## Dependencies

- `bootstrap.database`

## Validation

- Run: `npm run build`
- Manual: Backend starts and /api/health or /admin responds
